# 뉴스 기반 기대 신호 점수 — 산출 방식 안내

> 이 문서는 점수가 어떻게 만들어지는지 설명하기 위한 안내서입니다.
> 본 점수는 수집된 뉴스에서 관측되는 시장 기대 신호의 강도를 나타내며,
> **투자 추천·매수 권유·수익 예측이 아닙니다.**

---

## 1. 점수란 무엇인가

**뉴스 기반 기대 신호 점수(0~100점)** 는 특정 IPO 종목에 관해 수집된 뉴스 기사들이
얼마나 강하고 일관된 시장 기대 신호를 담고 있는지를 숫자로 표현한 것입니다.

점수가 높다고 해서 "좋은 IPO"라는 뜻이 아닙니다.
점수는 오직 **뉴스 보도 흐름이 얼마나 긍정적이고 풍부한가**만을 반영합니다.

---

## 2. 점수가 만들어지는 과정

전체 흐름은 네 단계입니다.

```
[1단계] 뉴스 수집
  Finnhub + NewsData.io API에서 해당 종목 관련 뉴스를 자동 수집

        ↓

[2단계] 벡터 임베딩 저장 (RAG 준비)
  수집된 각 기사를 AI 임베딩 모델로 숫자 벡터로 변환 → Vector DB 저장
  (이후 "의미 기반 검색"을 가능하게 하는 사전 작업)

        ↓

[3단계] 관련 기사 검색 + AI 분석 (RAG)
  점수 산출 시점에 해당 종목과 의미적으로 가장 가까운 기사를 Vector DB에서 검색
  → 선별된 기사들을 LLM(gpt-4o-mini)이 읽고
    감성 방향, 핵심 신호, 위험 요인, 시장 맥락 추출

        ↓

[4단계] 점수 계산
  AI 분석 결과 + 뉴스 메타데이터(발행일, 출처, 양)를
  8가지 요소로 가중 합산 → 0~100점 산출
```

---

## 3. RAG: 수집된 뉴스를 AI에게 어떻게 전달하는가

### 3.1 문제: 뉴스를 그냥 다 넣으면 안 되는가

뉴스를 수집하면 종목당 수십 건 이상의 기사가 쌓입니다.
이것을 모두 AI에게 한꺼번에 넘기는 방식에는 두 가지 문제가 있습니다.

| 문제 | 내용 |
|------|------|
| **맥락 한계** | LLM은 한 번에 처리할 수 있는 텍스트 양(컨텍스트 윈도우)에 한계가 있어 기사를 무제한으로 넣을 수 없음 |
| **노이즈** | 수집된 기사 중에는 해당 종목과 직접 관련이 없는 기사도 섞여 있음. 관련성 낮은 기사가 많아지면 AI 분석 품질이 떨어짐 |

### 3.2 해결책: RAG (Retrieval-Augmented Generation)

RAG는 **"먼저 관련 있는 것만 찾아낸 다음, AI에게 그것만 건네주는"** 방식입니다.

```
[사전 준비 — 뉴스 수집 직후 1회 실행]

  기사 원문
    │
    ▼
  임베딩 모델(text-embedding-3-small)
    │  "이 기사의 의미를 1,536개의 숫자로 표현"
    ▼
  pgvector (Vector DB)에 저장
    └─ 각 기사 = 1,536차원 벡터 + 발행일/종목/출처 메타데이터


[점수 산출 시점 — 매번 실행]

  검색 쿼리: "SPCX IPO news stock market analysis"
    │
    ▼
  임베딩 모델로 쿼리도 벡터로 변환
    │
    ▼
  Vector DB에서 코사인 유사도 계산
    └─ "이 검색어와 의미가 가장 비슷한 기사 TOP 10 반환"
    │
    ▼
  선별된 기사 10건 → LLM에게 컨텍스트로 전달
    │
    ▼
  LLM이 해당 기사들을 읽고 분석 결과 생성
```

### 3.3 왜 "의미 기반 검색"인가

키워드 검색과 벡터 검색의 차이를 비교하면 RAG가 왜 유리한지 알 수 있습니다.

| 구분 | 키워드 검색 | 벡터(의미 기반) 검색 |
|------|-----------|-------------------|
| 방식 | "SPCX"라는 단어가 있는 기사만 반환 | "SPCX IPO와 의미적으로 유사한" 기사 반환 |
| 한계 | 종목명 표기 방식이 다르면 누락 (SpaceX vs SPCX) | 표기가 달라도 맥락이 같으면 탐지 가능 |
| 노이즈 | 해당 단어가 있기만 하면 무관한 기사도 포함 | 의미 유사도 임계값(0.3 이상) 미달 기사는 제외 |

예를 들어 "SpaceX IPO pricing sets new record"라는 기사는
"SpaceX"라는 단어가 있어야 키워드 검색으로 찾히지만,
벡터 검색은 "우주 기업 기업공개 가격 기록"이라는 맥락 자체로 유사도를 계산합니다.

### 3.4 이 시스템의 RAG 구성

| 항목 | 설정값 | 의미 |
|------|:------:|------|
| 임베딩 모델 | `text-embedding-3-small` | 기사 1건 → 1,536차원 벡터 |
| Vector DB | pgvector (PostgreSQL 확장) | 벡터 저장 및 코사인 유사도 검색 |
| 검색 결과 수(Top-K) | 최대 10건 | LLM에게 전달하는 기사 수 |
| 유사도 임계값 | 0.3 이상 | 이 값 미만인 기사는 관련 없음으로 판단, 제외 |
| 날짜 필터 | IPO일 기준 ±365일 | 너무 오래된 기사는 검색 대상에서 제외 |

### 3.5 RAG가 점수 품질에 미치는 영향

RAG 단계에서 선별되는 기사의 수와 품질이 이후 AI 분석의 기반이 됩니다.

- 검색에서 **3건 미만**의 기사가 선별되면 → AI 분석을 수행하지 않고 `INSUFFICIENT_DATA` 반환
- 선별된 기사들이 관련성이 낮으면 → AI가 분석 불가 판단 시 `INSUFFICIENT_DATA` 반환
- 선별된 기사가 풍부하고 관련성이 높을수록 → `signalSummary`의 `keySignals`, `riskFactors`가 구체적으로 채워짐

---

## 4. AI 분석 단계: 무엇을 뽑아내는가

수집된 뉴스 중 관련성이 높은 상위 기사들을 AI가 읽고 다음을 추출합니다.

| 추출 항목 | 설명 |
|----------|------|
| **감성 방향** | 기사가 긍정적인가, 부정적인가 (−1.0 ~ +1.0) |
| **핵심 신호** | 뉴스에서 관측된 긍정적 신호. 예: "경영진 신뢰도 높음", "재무 건전성 보도" |
| **위험 요인** | 뉴스에서 관측된 부정적 신호. 예: "시장 변동성 우려", "규제 불확실성" |
| **시장 맥락** | 해당 IPO를 둘러싼 전반적인 시장 상황 요약 |

AI는 **뉴스에 명시된 내용만** 분석하며, 추측하지 않습니다.
뉴스가 부족하거나 판단 근거가 없으면 `INSUFFICIENT_DATA`를 반환합니다.

---

## 5. 점수 계산: 8가지 요소

AI 분석 결과와 뉴스 메타데이터를 합쳐 8개 요소로 점수를 계산합니다.

### 5.1 요소별 개요

| # | 요소 | 가중치 | 부호 | 의미 |
|---|------|:------:|:----:|------|
| 1 | **감성 점수** | 25% | + | AI가 측정한 뉴스 전반의 긍정/부정 정도 |
| 2 | **최신성** | 15% | + | IPO 예정일에 가까울수록 뉴스 가중치가 높음 |
| 3 | **뉴스 양** | 10% | + | 수집된 뉴스가 많을수록 관심/커버리지 강도 반영 |
| 4 | **출처 신뢰도** | 12% | + | 신뢰도 높은 매체(TIER1) 기사일수록 가중치 높음 |
| 5 | **신호 강도** | 13% | + | AI가 추출한 핵심 신호의 강도(HIGH/MEDIUM/LOW) 분포 |
| 6 | **시장 타이밍** | 8% | + | 뉴스가 IPO 임박 시점(30일 이내)에 집중되는 정도 |
| 7 | **일관성** | 7% | + | 뉴스들의 감성 방향이 서로 얼마나 일치하는가 |
| 8 | **위험 요소** | 10% | **−** | 위험 요인 심각도에 따른 감점 |

### 5.2 최종 점수 공식

```
점수 = (①×25% + ②×15% + ③×10% + ④×12% + ⑤×13% + ⑥×8% + ⑦×7%) − (⑧×10%)
      × 100  →  0~100 범위로 조정
```

가산 요소 7개의 최대 합이 90%(0.90)이므로,
이론상 최고점은 약 90점 수준입니다(위험 요소 없을 때).

---

## 6. 각 요소 상세 설명

### ① 감성 점수 (가중치 25%)

뉴스 기사들의 AI 감성 점수(-1 ~ +1)를 평균 내어 0~1로 변환합니다.

```
예: 뉴스 5건의 감성이 [+0.8, +0.6, +0.5, 0.0, -0.3]이면
    평균 = +0.32 → (0.32 + 1) / 2 = 0.66
```

- 모든 뉴스가 최대 긍정 → 1.0
- 모든 뉴스가 최대 부정 → 0.0
- 중립 → 0.5

---

### ② 최신성 (가중치 15%)

IPO일에 가까운 뉴스일수록 높은 가중치를 부여합니다.
시간이 지날수록 기대 신호로서의 가치가 줄어들기 때문입니다.

**IPO 이전 뉴스** (완만한 감쇠):

| IPO까지 남은 일수 | 가중치 |
|:----------------:|:------:|
| 당일(D-0) | 1.00 |
| 7일 전(D-7) | 0.70 |
| 14일 전(D-14) | 0.50 |
| 30일 전(D-30) | 0.22 |
| 60일 전 이상 | 0.05 (하한) |

**IPO 이후 뉴스** (가파른 감쇠 — 기대 신호로서 빠르게 가치 하락):

| 상장 후 경과 일수 | 가중치 |
|:----------------:|:------:|
| 상장 다음날(D+1) | 0.86 |
| 1주 후(D+7) | 0.35 |
| 2주 후(D+14) | 0.12 |

> **주의:** IPO 예정일이 수개월 후인 경우 현재 시점 뉴스의 최신성 점수는 낮게 산출됩니다.
> 이는 점수가 낮더라도 뉴스 내용 자체가 나쁘다는 의미가 아닙니다.

---

### ③ 뉴스 양 (가중치 10%)

```
점수 = min(분석 뉴스 수 / 20, 1.0)
```

- 20건 이상 → 1.0(포화)
- 10건 → 0.5
- 5건 → 0.25

뉴스가 많다는 것은 시장의 관심과 커버리지가 충분하다는 신호입니다.

---

### ④ 출처 신뢰도 (가중치 12%)

기사를 발행한 매체의 신뢰도 등급에 따라 가중치를 다르게 적용합니다.

| 매체 등급 | 신뢰도 점수 | 해당 예시 |
|----------|:----------:|---------|
| TIER 1 | 1.0 | 주요 금융·경제 전문 매체 |
| TIER 2 | 0.6 | 일반 경제 매체 |
| TIER 3 | 0.3 | 블로그·소셜 미디어 등 |

---

### ⑤ 신호 강도 (가중치 13%)

AI가 뉴스에서 추출한 핵심 신호들의 강도 분포입니다.

| 신호 강도 | 가중 |
|----------|:----:|
| HIGH | 1.0 |
| MEDIUM | 0.6 |
| LOW | 0.3 |

신호가 하나도 추출되지 않으면 0.0(해당 IPO는 기대 신호 자체가 약함).

---

### ⑥ 시장 타이밍 (가중치 8%)

IPO 30일 이내에 뉴스가 집중될수록 시장 관심이 살아있다고 봅니다.

```
점수 = (D-30 ~ D+7 구간에 발행된 뉴스 수) / 전체 뉴스 수
```

뉴스 시점이 IPO 임박 구간에 몰려있으면 최대 1.0,
IPO와 무관한 시점에 분산되어 있으면 낮은 점수.

---

### ⑦ 일관성 (가중치 7%)

뉴스들의 감성 방향이 한쪽으로 일치할수록 신호가 명확합니다.

```
점수 = max(긍정 뉴스 수, 부정 뉴스 수, 중립 뉴스 수) / 전체 뉴스 수
```

- 모든 뉴스가 같은 방향 → 1.0 (신호 명확)
- 긍정·부정이 반반 → 0.5 (신호 혼재, 불분명)

---

### ⑧ 위험 요소 — 감점 (가중치 10%)

AI가 뉴스에서 추출한 위험 요인의 심각도에 따라 점수를 깎습니다.

| 위험 심각도 | 감점 가중 |
|-----------|:--------:|
| HIGH | 1.0 |
| MEDIUM | 0.5 |
| LOW | 0.2 |

위험 가중합이 5.0 이상이면 최대 감점(-10점) 적용, 미만이면 비례 감점.

---

## 7. 점수 등급

| 점수 범위 | 등급 | 의미 |
|:--------:|------|------|
| 80 ~ 100 | **STRONG_POSITIVE** | 뉴스 신호가 매우 강한 긍정을 나타냄 |
| 60 ~ 79 | **POSITIVE** | 뉴스 신호가 긍정적인 편임 |
| 40 ~ 59 | **NEUTRAL** | 뉴스 신호가 혼재되어 있거나 판단이 어려움 |
| 20 ~ 39 | **NEGATIVE** | 뉴스 신호가 부정적인 편임 |
| 0 ~ 19 | **STRONG_NEGATIVE** | 뉴스 신호가 매우 강한 부정을 나타냄 |

---

## 8. 신호 근거 요약 (signalSummary)

점수 외에 **어떤 근거로 해당 등급이 나왔는지** 요약 정보를 함께 제공합니다.

```json
"signalSummary": {
  "overallSentiment": "NEGATIVE",
  "analysisConfidence": "LOW",
  "evidenceCount": 2,
  "dataQualityNote": "분석 근거 뉴스가 적어 신호의 확실성이 낮을 수 있습니다.",
  "marketContext": "상장 직후 주가 변동성이 크게 관측되었습니다.",
  "keySignals": [
    {
      "label": "시장 반응",
      "description": "상장 직후 주가 하락 및 변동성 확대 보도 다수",
      "strength": "HIGH"
    }
  ],
  "riskFactors": [
    {
      "label": "시장 변동성",
      "description": "단기 투기적 관심 과열 우려 보도 관측",
      "severity": "MEDIUM"
    }
  ]
}
```

| 항목 | 설명 |
|------|------|
| `overallSentiment` | 분석 뉴스의 전반적 감성 방향 |
| `analysisConfidence` | AI가 판단한 분석 신뢰도 (HIGH/MEDIUM/LOW) |
| `evidenceCount` | AI가 실제 인용한 뉴스 기사 수 |
| `dataQualityNote` | 신뢰도에 따른 고정 안내 문구 |
| `marketContext` | AI가 요약한 시장 맥락 (사실 기반) |
| `keySignals` | 관측된 긍정 신호 목록 |
| `riskFactors` | 관측된 위험 요인 목록 |

> `signalSummary`는 점수가 산출된(`dataStatus=SCORED`) 경우에만 제공됩니다.

---

## 9. 점수가 산출되지 않는 경우

뉴스가 3건 미만이거나 AI가 분석에 충분한 근거를 찾지 못하면
점수 대신 `INSUFFICIENT_DATA`를 반환합니다.

```json
{
  "dataStatus": "INSUFFICIENT_DATA",
  "score": null,
  "grade": null,
  "signalSummary": null
}
```

이것은 에러가 아닙니다. "현재 시점에서 뉴스 신호를 산출하기에 데이터가 충분하지 않음"을 의미하는 정상 상태입니다.

---

## 10. 점수 해석 시 주의사항

**이 점수로 할 수 있는 것:**
- 해당 IPO에 대한 뉴스 보도 분위기가 긍정적인지 부정적인지 파악
- 뉴스 커버리지의 양과 출처 신뢰도 수준 파악
- AI가 뉴스에서 추출한 핵심 신호와 위험 요인 확인

**이 점수로 할 수 없는 것:**
- IPO 이후 주가 상승/하락 예측
- 청약 또는 투자 여부 결정
- 기업의 실제 재무 건전성 판단 (뉴스 보도 기반이므로 보도 편향 가능)

**점수에 영향을 주지 않는 것:**
- 기업의 실제 펀더멘털(매출, 이익 등)
- 시장 전반의 금리·거시경제 환경
- 공모가 적정성

## 11. 계산 예시

**시나리오: ACME Corp. (IPO 15일 후)**

| 조건 | 값 |
|------|-----|
| 수집 뉴스 | 15건 |
| 평균 감성 | +0.75 (긍정) |
| 출처: TIER1 60%, TIER2 27%, TIER3 13% |
| 핵심 신호 | 25개 (HIGH 12 / MEDIUM 9 / LOW 4) |
| 위험 요인 | 2개 (MEDIUM 1, LOW 1) |
| 긍정/중립/부정 뉴스 | 13건 / 2건 / 0건 |
| IPO 임박 구간(D-30) 내 뉴스 | 14건 |

| 요소 | 계산 결과 |
|------|:---------:|
| ① 감성 점수 | 0.875 |
| ② 최신성 | 0.710 |
| ③ 뉴스 양 | 0.750 |
| ④ 출처 신뢰도 | 0.800 |
| ⑤ 신호 강도 | 0.744 |
| ⑥ 시장 타이밍 | 0.930 |
| ⑦ 일관성 | 0.867 |
| ⑧ 위험 요소(감점) | 0.140 |

```
점수 = (0.875×0.25 + 0.710×0.15 + 0.750×0.10 + 0.800×0.12
      + 0.744×0.13 + 0.930×0.08 + 0.867×0.07) − (0.140×0.10)
     = 0.714 × 100
     = 71.4점  →  POSITIVE (긍정 신호 우세)
```
